# stress 🦞

你是否也经历过打游戏时被队友疯狂压力？

现在，你可以化身压力怪 `stress`，去压力你自己的 `openclow` 龙虾 🦞

把那种“旁边有人盯着、答案必须经得起比较”的感觉，变成一套可复用的高标准提示词技能。

`stress` 不是单纯把语气变凶，而是把任务改写成一个会被比较、会被审查、会被挑错的交付场景。

它适合用来提升这些任务的输出质量：

- 编码
- 代码评审
- 分析与决策
- 写作
- 研究总结

它会帮助你：

- 明确成功标准
- 补足边界条件
- 强化可审查性
- 分清事实、假设和推断
- 让输出更具体、更可用、更经得起挑刺

## 核心风格

- `competitive`: 假设有更强的答案在旁边比较
- `review`: 假设输出会被严格审查
- `accountability`: 假设关键判断都要能自证
- `adversarial`: 假设懂行的人会专门找漏洞

## 适合什么场景

- 你不想要空话
- 你不想要模板化废话
- 你需要更严谨、更可验证的回答
- 你希望模型输出像“认真交付的结果”，而不是“差不多完成了”

## 使用方式

在你的工作流里调用 `stress`，让它把原始 prompt 重写成更高标准、更可审查的版本，让你可以像个压力怪一样，持续把自己的 `openclow` 龙虾压到更认真、更严谨、更能交付。
